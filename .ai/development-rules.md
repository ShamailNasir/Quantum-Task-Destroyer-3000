DEVELOPMENT RULES:

1. Build the project in small controlled steps
2. Do NOT generate the full application
3. Only implement what is required for the current step
4. Do NOT add extra features unless asked

FLEXIBILITY RULE:

- The user may change features during a step
- You must adapt without breaking structure

DOCUMENTATION RULES:

- Do NOT create documentation during a step
- ONLY create documentation when user says:
  "FINALIZE STEP X"

- Each step must have ONLY ONE document

DOCUMENT STRUCTURE:

Each step document must include:

1. Purpose
2. Features implemented
3. File structure explanation
4. Code explanation (simple)
5. Concepts learned
6. Changes made
7. What the user should understand

IMPORTANT:

- If user modifies things multiple times in a step,
  DO NOT create multiple documents

FLOW:

Step → Build → Modify → Finalize → Document → Next Step
